Service (extending)

A service is a distinct part of the functionality that is provided by an entity through interfaces. FeatureServer is based on a plugin architecture which invokes the requested service at runtime. As described in API Basics (RESTful) the to be invoked services follows after the base url as first parameter (see line 1) or could be set as SERVICE paramter (see line 2) as depict in the following example: 

http://featureserver.org/wfs/
http://featureserver.org?service=WFS
FeatureServer tries to find the service name, where the pattern in line 1 would be preffered if both patterns are combined, and automatically invokes the service class in the module featureserver.services.{service}. Hence, if you want to add a new service simple do the following:
  1. Create a new file in the folder featureserver/services called {service}.py (have to be lowercase).
  2. Define a new class in upper case which is called as your service.
  3. Import the service interface from service import Service
  4. Overwrite the method def parse(self): pass
When FeatureServer detectes the serivce, it imports the right module and instatiates the servcie class. Then it calls the parse parameter which is resonsible for the service specific request handling.

DataSource (extending)

A data source holds all the data which could be requested by a service. In case of service=WFS a data source stores geografic features such as points, polygons, and so on. API Basics (RESTful) defines the usage of layers / type names. Such a layer need to be set up in the configuration file of the FeatureServer, where the name in the brakcets [roads] is the name of the layer and each layer need to have the attribute type={DataSource} which defines the data source which should be invoked at runtime when requesting the layer. The name the data source module and class should have the same name as the value defined in the type={DataSource} respecting cases. Hence, to add a new data source do the following:

  1. Create a new file in the folder featureserver/datasource called code class="prettyprint">{DataSource}.py.
  2. Define a new class which is called as your data source.
  3. Import the data source interface from DataSource import DataSource
  4. Overwrite the needed methods.

OutputFormat (extending)

Output formats transform data from one format to an other using for example the VectorFormats library. A requested format (see API Basics (RESTful)) will be automatically invoked at runtime by the FeatureServer. Important is that each output format implements the interface OutputFormat located in the module featureserver.output_format.OutputFormat. To add a new output format simple do the following:

  1. Create a new file in the folder featureserver/output_format called {OUTPUTFormat}.py (have to be uppercase).
  2. Define a new class in upper case which is called as your output format.
  3. Import the output format interface from OutputFormat import OutputFormat
  4. At least overwrite the method def encode(self): pass
