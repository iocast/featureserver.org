Upcoming 2.0 up

  • Multi-layer support as defined in WFS 1.1.0 and 2.0.0.
  • Addtional configuration key to indicate if requested FE attributes should be included into result list fe_attributes=[true|false]
  • Addtional configuration key for Twitter and Flickr. Key attributes defines which values should be included in the response.
  • PostGIS became PostGIS and PostGISHstore where PostGISHstore is derived from PostGIS
  • Renamings / Definitions:
    • Servcie was renamed to OutputFormat
    • Vector format and output format 'SQLite' was renamed to SpatialLite
    • Service became the same definition as descried by OGC. Each servcie can be called as the first parameter in the URL e.g. http://featureserver.org/{service}/... or a service attribute need to be set in the GET parameters or POST payload which should be parsable.
  • Several bug fixes:
    • Twitter datasource uses now Twitter API 1.1 instead of the twittervision API
  • Miscellaneous:
    • Namespace changed to "http://featureserver.org/fs"

R 1.15.2 up

  • Bug fix: GeoAlchemy bounding box support. bbox is defined as follow: left, bottom, top, right.

R 1.15.1 up

  • Bug fix: GeoAlchemy operator such as like, ilike, etc. works again.

Download here

R 1.15 up

  • All new exception/error handling:
    Returning error messages in format of the requested service if it is supported. Else it takes the default format if supported, WFS/GML otherweise.
    • CSV, GeoJSON, GeoRSS, GML/WFS, HTML
    • Addtional key default_exception (optional) indicates the service to be used if an error occures.
  • Several bug fixes:
    • OSM service now supports LineString
    • GPX service does not crash anymore if elevation attribute is not given (ele is now an optional parameter)
    • Every parameter in FE is converted to string beforehand
    • GPX supports LineString and Polygon. Polygons are converted to LineString and then treated as LineString.

Download here

  • Addtional Service:
    • DXF (AutoCAD)
  • Additional Datasource:
    • SpatialLite is an open source library intended to extend the SQLite core to support fully fledged Spatial SQL capabilities.
  • WFS-T 1.1.0 support for datasource SpatialLite.

R 1.13 up

  • WFS-T 1.1.0 support for datasource PostGIS.
  • Addtional Services:
    • GPX a light-weight XML-based GPS Exchange Format for the interchange of waypoints, routes, and tracks.
    • SQLite/SpatiaLite is an open source library intended to extend the SQLite core to support fully fledged Spatial SQL capabilities.
    • Shapefile is a popular geospatial vector data format for geographic information systems software developed and regulated by Esri.
    • OV2 is TomTom's points of interest database.
  • hstore support for datasource PostGIS.
